## What Is a Teamspace?

A Teamspace is a shared namespace within an Organization. Think of it as a project folder with its own members list, storage bucket, and compute budget. Most day-to-day work — creating Studios, launching jobs, storing artifacts — happens inside a Teamspace.

## Teamspace Isolation

Resources are scoped to a Teamspace. A Studio or dataset in Teamspace A is not visible to Teamspace B unless explicitly shared. This isolation makes Teamspaces useful for:

- Separating environments (development, staging, production).
- Running independent research projects under the same billing account.
- Granting external collaborators access to a specific project without exposing the whole Organization.

## Managing Teamspace Settings

Teamspace admins can configure the following from **Teamspace → Settings**:

- **Name and visibility** — rename or archive the Teamspace.
- **Storage** — set a storage quota and view current usage.
- **Compute budget** — cap the credits a Teamspace can consume per month.
- **Members** — add or remove members independently of the Organization member list. For more information, see [Members](/concepts/members).
