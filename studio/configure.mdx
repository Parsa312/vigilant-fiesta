## Studio types

When you create a Studio, you first select a Studio type. Each type comes
pre-configured for a specific workflow.

| Studio type     | Description                                                                                                                              |
|-----------------|------------------------------------------------------------------------------------------------------------------------------------------|
| AI development  | General-purpose coding environment. Connect from your local IDE, train models, and more.                                                |
| Python          | Specialized for Python development with faster cold starts.                                                                              |
| ComfyUI         | Ready-to-use environment for generating images, videos, and audio.                                                                      |
| Streamlit App   | Build and iterate on interactive Streamlit apps instantly.                                                                               |
| OpenClaw        | Deploy OpenClaw on a Studio.                                                                                                             |
| Prep data       | Optimized for downloading and processing datasets. For distributed data processing, install the Data Preparation plugin.                |
| Web app         | Develop web apps using React templates and the ports plugin.                                                                             |
| Machine learning| Pre-configured with Streamlit and Gradio examples.                                                                                      |

## Choosing a machine

After selecting a Studio type, configure the underlying compute. The machine
selector has three tabs: CPU, GPU, and TPU.

### CPU

CPU instances are suitable for development, prototyping, and lightweight data
tasks that don't require GPU acceleration.

| CPUs          | Model         | Speed (GHz) | Memory (GB) |
|---------------|---------------|-------------|-------------|
| 4 / 8 / 16    | Default (CPU) | 2.00        | 16          |
| 32 / 64 / 96  | Data prep     | 2.00        | 128         |

The Default (CPU) instance is free. Data prep instances are paid. Current
pricing is available in the machine selector.

### GPU

GPU instances are suitable for model training, fine-tuning, and inference.
You select both the GPU model and the number of units.

| Model     | Speed (TFLOPs) | Memory (GB) | CPUs |
|-----------|----------------|-------------|------|
| T4        | 125            | 16          | 8    |
| L4        | 121            | 24          | 8    |
| L40S      | 362            | 48          | 16   |
| RTXP 6000 | 500            | 96          | 48   |
| A100      | 312            | 40          | 30   |
| H100      | 1979           | 80          | 24   |
| H200      | 1979           | 141         | 16   |

Most models support 1, 2, 4, or 8 units. T4 supports 1 or 4 units; H200
supports 1 or 8. Current pricing is available in the machine selector.

### TPU

TPU instances are suitable for large-scale training workloads that benefit
from tensor processing acceleration. Currently one TPU option is available.

| Model | Speed (TFLOPs) | Memory (GB) | Cores       |
|-------|----------------|-------------|-------------|
| TPU   | 918            | 32          | 1 / 4 / 8   |

Note that TPU instances have a longer average wait time than CPU or GPU
instances.

## Interruptible instances

All machine types support an interruptible option. Enabling it allows
Lightning to pause your Studio to free up capacity, at a lower cost than
a standard instance.

Recent in-memory changes may not be saved if a Studio is interrupted, but
your persistent environment — installed packages, files, and configuration
— is preserved.

H200 instances are not available as interruptible.
