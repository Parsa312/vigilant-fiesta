Most AI inference tools are built around single-model APIs with rigid abstractions that get in the way when your pipeline grows more complex. LitServe takes a different approach: you write full AI pipelines — not just model wrappers — in clean, extensible Python. Built on FastAPI and optimized for AI workloads, it gives you multi-model serving, streaming, batching, and custom inference logic from a single server.

## Install

```bash
pip install litserve
```

## Define your server

Subclass `LitAPI` and implement four methods:

```python
import litserve as ls

class SimpleLitAPI(ls.LitAPI):
    def setup(self, device):
        self.model1 = lambda x: x**2
        self.model2 = lambda x: x**3

    def decode_request(self, request):
        return request["input"]

    def predict(self, x):
        a = self.model1(x)
        b = self.model2(x)
        c = a + b
        return {"output": c}

    def encode_response(self, output):
        return {"output": output}

if __name__ == "__main__":
    server = ls.LitServer(SimpleLitAPI(max_batch_size=1), accelerator="auto")
    server.run(port=8000)
```

`setup` loads your models onto the target device. `decode_request` converts the incoming request payload into the input your model expects. `predict` runs inference — you can call multiple models, chain outputs, or apply any custom logic here. `encode_response` converts the prediction output into the response returned to the caller.

## Deploy

```bash
lightning deploy server.py --cloud
```

LitServe handles batching, streaming, device selection, and endpoint creation automatically.

## What LitServe supports

- Any model type: LLMs, diffusion models, classical models
- Multi-model serving
- Batching
- Streaming
- Custom inference logic
