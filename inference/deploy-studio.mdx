---
title: "Deploy a Studio"
description: "Turn your working Studio into a production-grade server — no Docker, no YAML."
---

Snapshot Deploy turns your working Studio into a real deployment without requiring you to package a container or write infrastructure config. Lightning snapshots the full Studio environment — code, installed packages, exposed ports — and launches it as a scalable, monitored deployment.

## Deploy via the UI

### Initial deployment

1. Install the Deployment plugin in your Studio.
2. Click **New Deployment** and select **Studio** as the source.
3. Enter the command you use to run your server.
4. Click **Deploy**.

Lightning handles packaging the environment, launching the deployment, and managing traffic, autoscaling, logging, and cold starts.

[Watch: Create a deployment from a Studio snapshot](https://pl-bolts-doc-images.s3.us-east-2.amazonaws.com/app-2/studio_deploy.mp4)

### Iterative re-deploy

Once a deployment is live, you can continue developing in the Studio and push updates without changing environments.

1. Open the Deployment plugin and select your existing deployment.
2. Click **Update** in the top right.
3. Review the settings and click **Update** to confirm.

Lightning packages the new Studio environment, re-deploys using a rollout strategy, and automatically reverts if something goes wrong.

[Watch: Update a deployment to the latest Studio version](https://pl-bolts-doc-images.s3.us-east-2.amazonaws.com/Docs_Studios_MP4/Deploy-StudioSnapshot-Deployiterative.mp4)

### Revert versions

1. Open the Deployment plugin and select your deployment.
2. Click the **Releases** tab.
3. Click the three-dot menu on the release you want to restore.
4. Confirm the restoration.

[Watch: Restore an earlier release](https://pl-flash-data.s3.us-east-1.amazonaws.com/restore-any-releases.mp4)

## Deploy via the SDK

### Single server

Run this code from within a Studio to deploy the current environment as a production service:

```python
from lightning_sdk import Studio, Deployment
from lightning_sdk.deployment import HttpHealthCheck

deployment = Deployment()
deployment.start(
    studio=Studio(),
    command="python -m http.server",
    ports=[8000],
    health_check=HttpHealthCheck(
        path="/",
        port=8000
    )
)
```

To create a Studio, upload code to it, and deploy — all programmatically:

```python
from lightning_sdk import Studio, Deployment
from lightning_sdk.deployment import HttpHealthCheck
from datetime import datetime

server_code = """
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"Hello": "World"}
"""

studio_name = "studio-" + datetime.now().strftime("%m-%d_%H:%M:%S")
studio = Studio(studio_name)
studio.start()

studio.run_with_exit_code("pip install 'fastapi[standard]'")

with open("server.py", "w") as f:
    f.write(server_code)
studio.upload_file("server.py")

deployment = Deployment()
deployment.start(
    studio=studio,
    command="fastapi run server.py",
    ports=[8000],
    health_check=HttpHealthCheck(
        path="/",
        port=8000
    )
)

resp = deployment.get("/")
assert resp.json() == {"Hello": "World"}
```

### Multiple servers

To run multiple servers concurrently from the same deployment:

```python
from lightning_sdk import Studio, Deployment

commands = [
    "python server_1.py --port 3000 &",
    "python server_2.py --port 3001 &",
]

deployment = Deployment()
deployment.start(
    studio=Studio("your-studio-name"),
    commands=commands,
    ports=[3000, 3001],
)
```

### Update secrets

Add a secret and environment variable to an existing deployment:

```python
from lightning_sdk.deployment import Deployment, Secret, Env

deployment = Deployment("existing-deployment")
deployment.update(env=[Secret(name="my-secret"), Env(name="env-name", value="env-value")])
```

Copy environment configuration from one deployment to another:

```python
from lightning_sdk.deployment import Deployment

source_deployment = Deployment("source-existing-deployment")
target_deployment = Deployment("target-existing-deployment")

target_deployment.update(env=source_deployment.env)
```
