Use Firecrawl to get web data into an accessible format for AI agents and LLMs. You can use the Firecrawl API to:

- Scrape any page
- Crawl an entire site

 The API takes care of:

- Headless browser rendering, including JavaScript-rendered pages that a plain HTTP request would return empty
- HTML parsing
- Rate-limit handling

In return, you get clean data back in the following formats:

- LLM-ready Markdown
- Structured JSON
- Screenshots

<Note>
  This is an independent documentation sample created by Parsa Shahbodaghi to demonstrate an approach to Firecrawl's developer docs, verified against the live API. It is not official Firecrawl documentation and is not affiliated with or endorsed by Firecrawl.

  Testing surfaced one real finding: Firecrawl's documented keyless mode — calling `Firecrawl()` with no API key, which caps you at lower rate limits than an authenticated call — works correctly in the REST API and the Node SDK. However, the Python SDK throws `ValueError: No API key provided` under the same conditions. Every code example in this sample reads the API key from an environment variable rather than showcasing keyless access.
</Note>

## Core concepts

- **Scrape** — fetch a single URL and get back Markdown, HTML, links, structured data, or a screenshot.
- **Crawl** — start from a URL, follow links across a site, and scrape every page that matches your filters. Crawls run asynchronously; you poll a status endpoint until they finish.

Both endpoints accept the same `formats` and extraction options, so what you learn scraping one page applies directly to crawling a whole site.
