This guide covers installing `firecrawl-py` and using it to scrape and crawl pages. Before you start, you'll need a Firecrawl API key stored as the `FIRECRAWL_API_KEY` environment variable. For more information, see [Quickstart](/quickstart) if you haven't set one up yet.

## Install

Install the firecrawl-py package using the following command:

```bash
pip install firecrawl-py
```

## Authenticate

Create a `Firecrawl` client using the API key you stored as an environment variable:

```python
import os
from firecrawl import Firecrawl

app = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])
```

## Scrape a page

Call `scrape()` with a URL and the formats you want back:

```python
result = app.scrape(
    "https://firecrawl.dev",
    formats=["markdown", "links"],
    only_main_content=True,
)

print(result.markdown)
print(result.metadata.title)
print(result.links[:5])
```

`scrape()` is synchronous — it blocks until the page has been rendered and returns the requested formats directly, with the same fields documented in the [`/v2/scrape` reference](/api-reference/scrape).

## Crawl a site

Call `crawl()` with a starting URL to scrape it and its linked pages:

```python
job = app.crawl(
    "https://docs.firecrawl.dev",
    limit=50,
    scrape_options={"formats": ["markdown"]},
)

print(job.status)         # "completed"
print(job.completed, "/", job.total)

for page in job.data:
    print(page.metadata.source_url, "->", len(page.markdown), "chars")
```

`crawl()` starts the job and polls `GET /v2/crawl/{id}` under the hood, returning once the crawl reaches a terminal state. If you'd rather manage polling yourself — for example, to update a progress bar — start the job and poll manually:

```python
job = app.start_crawl("https://docs.firecrawl.dev", limit=50)

status = app.get_crawl_status(job.id)
while status.status == "scraping":
    print(f"{status.completed}/{status.total} pages")
    time.sleep(3)
    status = app.get_crawl_status(job.id)

print(status.status)
```
